## Update: 12:20 PM - Gog CLI Investigation

Tried using `gog` CLI to bypass OAuth issues with Google Sheets.

### Findings
- **Gog is installed** at `/usr/local/bin/gog`
- **Has existing tokens** in `~/.config/gogcli/tokens.json`
- **BUT**: Uses old Desktop OAuth client: `391258212025-nafc7rponmj6i0ot2326j8ontcete0gn.apps.googleusercontent.com`
- **Result**: Same "invalid_client" / OAuth blocked error

### Attempted Commands
```bash
gog auth add mat@craftable.com --services sheets  # No auth for sheets
gog sheets get [sheet-id] [range]                  # Requires OAuth auth
```

### Conclusion
Gog CLI cannot bypass the OAuth issue because it's configured with the old Desktop app credentials that Google now blocks/restricts.

### Recommended Path Forward
**Service Account approach** - Generate JSON key, share sheet once, done forever. No OAuth flows, no token expiration, no "invalid_client" errors.
